# Security policy

## Supported versions

Security fixes are applied to the latest `1.x` release and `main`.

| Version | Supported |
| --- | --- |
| Latest `1.x` | Yes |
| `0.x` | No |

## Reporting a vulnerability

Do not open a public issue for credentials exposure, authentication bypass,
unsafe file handling, or another exploitable vulnerability.

Use GitHub's **Report a vulnerability** action in the repository Security tab.
If private vulnerability reporting is unavailable, contact the maintainer
through the GitHub profile and request a private channel without including
exploit details in the first public message.

Include the affected version/commit, Apple platform and OS version, impact,
minimal reproduction, and any suggested mitigation. Remove real stream tokens,
cookies, user data, FairPlay certificates, SPC/CKC payloads, and license-server
credentials.

The maintainer will acknowledge a complete report as soon as practical,
coordinate validation privately, and publish remediation notes after a fix is
available. No fixed response-time or bounty program is promised.
